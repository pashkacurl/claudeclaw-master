# Аудит веб-сайтов

## 📦 Установка
1. Скопируйте папку `audit-website/` в `skills/audit-website/` вашего workspace
2. OpenClaw подхватит автоматически
3. Проверка: "проверь сайт example.com"

## 🎯 Что умеет

Проверка сайта по 3 категориям (UX-фокус):
- **Контент и UX:** читаемость, CTA, value proposition, навигация
- **Доступность (a11y):** alt-текст, семантика, контраст, ARIA, клавиатурная навигация
- **Мобильная версия:** адаптивность, touch targets, шрифты

> **SEO, производительность, безопасность, schema → используй скил `/seo`** (seo-page, seo-technical, seo-schema). Там полный набор из 18 субскилов.

Выдаёт:
- Оценку 0-100 баллов
- Приоритизированный список проблем (критичные → важные → рекомендации)
- Конкретные решения для каждой проблемы

## ⚙️ Как работает

### Процесс (4 этапа)

#### 1. Сбор данных

Для каждого URL выполняет:

**Через web_fetch (быстро, базовый анализ):**
```
web_fetch(url) → контент, мета-теги, структура HTML
```

**Через браузер (для JS-heavy сайтов):**
```
browser(action=open, targetUrl=URL)
browser(action=snapshot) → структура DOM, интерактивные элементы
browser(action=screenshot) → визуальный анализ
```

**Дополнительно:**
- Проверка robots.txt и sitemap.xml
- Проверка скорости через время загрузки
- Анализ размера страницы и ресурсов

#### 2. Категории проверки

##### Контент и UX (влияет на конверсию)

**Чеклист:**
- [ ] **Value proposition:** чёткий в первом экране
- [ ] **CTA:** виден без скролла, контрастный
- [ ] **Мобильная адаптация:** responsive design
- [ ] **Читаемость:**
  - Размер шрифта ≥ 16px на мобильных
  - Контраст ≥ 4.5:1
  - Длина строки 50-75 символов
- [ ] **Нет битых ссылок:** все ссылки работают
- [ ] **Контактная информация:** легко найти
- [ ] **Скорость взаимодействия:** формы работают быстро
- [ ] **Понятная навигация:** меню интуитивное

**Типичные проблемы:**
- CTA в конце длинной страницы
- Непонятное предложение ("Мы делаем лучшие решения")
- Мелкий шрифт на мобильных
- Битые ссылки на старые страницы

##### Доступность (a11y - доступность для всех)

**Чеклист:**
- [ ] **Alt-текст:** у всех значимых изображений
- [ ] **Семантическая разметка:** `<nav>`, `<main>`, `<footer>`, `<article>`
- [ ] **Контраст текста:** ≥ 4.5:1 для обычного, ≥ 3:1 для крупного
- [ ] **Фокус-индикаторы:** видны при навигации с клавиатуры
- [ ] **ARIA-метки:** где нужно (кнопки-иконки, поля форм)
- [ ] **Язык документа:** `<html lang="ru">`
- [ ] **Skip links:** "Перейти к контенту" для скринридеров
- [ ] **Формы:** labels связаны с inputs

**Типичные проблемы:**
- Кнопки без текста (только иконки)
- Низкий контраст текста
- Отсутствие alt-текста
- Формы без labels

#### 4. Скоринг

**Система оценки:**

| Балл | Значение | Действия |
|------|----------|----------|
| 90-100 | 🟢 Отлично | Мелкие улучшения |
| 70-89 | 🟡 Хорошо | Есть что улучшить |
| 50-69 | 🟠 Средне | Нужна работа |
| <50 | 🔴 Плохо | Критичные проблемы |

**Вес проблем:**
- Критичная проблема: -10 баллов
- Важная проблема: -5 баллов
- Рекомендация: -2 балла

Стартовый балл: 100
Итоговый балл: 100 - (сумма штрафов)

## 🔧 Настройка под себя

### Кастомные правила

Создайте файл `data/audit-rules.md` с вашими стандартами:

```markdown
# Правила аудита

## SEO
- Title: 40-55 символов (строже чем дефолт)
- Обязательно: favicon.ico
- Обязательно: Google Analytics / Plausible

## Производительность
- Время загрузки: < 2 сек (строже)
- Размер страницы: < 1 MB (строже)
- Обязательно: CDN (Cloudflare/Fastly)

## Контент
- Обязательно: Live chat виджет
- Обязательно: Testimonials на главной
- CTA: минимум 2 на странице

## Ваши правила
[добавьте специфичные для вашей ниши]
```

### Приоритеты категорий

Настройте вес категорий под вашу цель:

**Лендинг (конверсия):**
```
Контент/UX: 50%
Доступность: 30%
Мобильная версия: 20%
```

**SaaS (UX + мобилка):**
```
Контент/UX: 40%
Мобильная версия: 35%
Доступность: 25%
```

### Шаблоны отчётов

Создайте шаблоны для разных типов сайтов:
- `templates/audit-ecommerce.md` - интернет-магазин
- `templates/audit-blog.md` - блог/медиа
- `templates/audit-saas.md` - SaaS продукт
- `templates/audit-landing.md` - лендинг

## 🔗 Разграничение со смежными скиллами

- **audit-website** (этот скил) = UX, доступность, конверсия, мобильная версия
- **`/seo audit`** = полный SEO-аудит (18 субскилов: техническое SEO, schema, контент, CWV, безопасность)
- **`/seo page`** = глубокий анализ одной страницы (on-page SEO)
- **`/seo technical`** = crawlability, indexability, security headers, Core Web Vitals

Для полного аудита рекомендуется запускать оба: audit-website (UX/a11y) + /seo audit (SEO/техника).
