# Brainstorming - Структурированный мозговой штурм

## 📦 Установка

1. Скопируйте папку `brainstorming/` в `skills/` вашего workspace
2. OpenClaw автоматически подхватит скилл при следующем сообщении
3. Проверка: скажите агенту "давай подумаем как лучше..."

## 🎯 Что умеет

- Превращать сырые идеи в структурированные планы
- Задавать уточняющие вопросы (по одному за раз)
- Предлагать 2-3 варианта решения с плюсами/минусами
- Фильтровать идеи через YAGNI (You Aren't Gonna Need It)
- Масштабировать детализацию под сложность задачи
- Фиксировать одобренные планы в памяти

## ⚙️ Как работает

### Железное правило

**НЕ переходить к реализации без одобренного плана.**

Даже если задача кажется простой - "простые" задачи часто содержат неочевидные решения.

### Процесс (5 шагов)

#### 1. Разведка контекста
- Проверить что уже есть (файлы, предыдущие решения)
- Заглянуть в `memory/` - может тема обсуждалась
- Понять ограничения: время, ресурсы, инструменты

**Агент выполняет:**
```bash
# Проверка существующих файлов
ls -la relevant_directory/

# Поиск в памяти
memory_search "ключевые слова темы"

# Проверка связанных скиллов
read SKILLS-MAP.md
```

#### 2. Уточняющие вопросы
- **По одному за раз** - не засыпать вопросами
- Предпочитать варианты выбора, а не открытые вопросы
- Фокус: цель, аудитория, критерии успеха, ограничения

**Примеры хороших вопросов:**
```
❌ "Расскажи подробнее про идею"
✅ "Это для личного использования или для клиентов?"

❌ "Какие требования?"
✅ "Важнее скорость разработки или гибкость системы?"

❌ "Что ты хочешь?"
✅ "Приоритет: A (автоматизация), B (аналитика) или C (визуализация)?"
```

#### 3. Предложить 2-3 подхода
- Каждый с плюсами и минусами
- Вести с рекомендацией и почему
- Быть конкретным - не абстракции, а конкретные шаги
- Применять YAGNI - убрать всё лишнее

**Формат вывода:**
```
💡 ТРИ ВАРИАНТА

Вариант 1: [Название] ⭐ (рекомендую)
Суть: [одно предложение]
Плюсы: [2-3 пункта]
Минусы: [1-2 пункта]
Время: [оценка]

Вариант 2: [Название]
Суть: [одно предложение]
Плюсы: [2-3 пункта]
Минусы: [1-2 пункта]
Время: [оценка]

Вариант 3: [Название]
Суть: [одно предложение]
Плюсы: [2-3 пункта]
Минусы: [1-2 пункта]
Время: [оценка]

🎯 РЕКОМЕНДАЦИЯ: Вариант 1 потому что [конкретная причина]

Какой выберешь?
```

#### 4. Оформить план реализации

**Главный принцип:** Пиши план так, будто его будет выполнять будущий-ты после рестарта — без контекста текущей сессии. Никаких "как обсуждали ранее" или "тот файл".

**Шапка плана (обязательно):**
```markdown
# [Название] - План реализации

**Цель:** Одно предложение - что получим в итоге
**Срок:** Оценка времени
**Зависимости:** Что нужно до начала (инструменты, доступы, данные)
```

**Каждый шаг — атомарный (2-10 минут):**
```markdown
### Шаг N: [Действие]

**Что делаем:** Конкретное описание
**Файлы:** Точные пути (создать/изменить/удалить)
**Команды:** Точные команды если есть
**Проверка:** Как убедиться что шаг выполнен
**Результат:** Что получаем после этого шага
```

**Гранулярность по типу задачи:**

| Тип | Шаг = | Детали |
|-----|-------|--------|
| Техническая | Одно действие | Точные пути, команды, ожидаемый вывод |
| Контентная | Один этап | Ссылки на скилы, референсы |
| Организационная | Одно действие | Контакты, адреса, дедлайны |

**Шкала детализации:**
```
Простая (< 30 мин):
"Создать файл X с Y контентом, сохранить в Z"

Средняя (1-3 часа):
1. Подготовка (15 мин)
2. Основная работа (1.5ч)
3. Проверка (30 мин)

Сложная (> 3 часа):
Разбить на фазы, утверждать каждую отдельно
```

**Правила написания планов:**
- Конкретные пути: ❌ "файл конфига" → ✅ "~/.openclaw/openclaw.json"
- Конкретные команды: ❌ "запусти тест" → ✅ "python3 scripts/test.py --verbose"
- Проверка каждого шага: ❌ "Создать файл" → ✅ "Создать файл → Проверка: ls -la file.md"
- YAGNI — только необходимые шаги, без "на всякий случай"
- DRY — если шаг повторяется, вынести в скрипт

Спросить одобрение перед переходом к реализации.

#### 5. Зафиксировать и запустить

**Куда сохранять:**
- Технический план → `memory/plans/YYYY-MM-DD-название.md`
- Контентный → в daily notes
- Разовый простой → прямо в чат, без файла

**Варианты запуска (предложить пользователю):**
1. **Сделать сейчас** — пошагово в этой сессии
2. **Субагент** — запустить автономное выполнение
3. **Крон** — если задача по расписанию
4. **Сохранить на потом** — записать и вернуться

Если план стоит сохранить — записать. Если контент-план — передать в соответствующий скил. Если автоматизация — зафиксировать в документации.

### Принципы работы

**Один вопрос = одно сообщение**
Не перегружать диалог. Лучше 3 последовательных вопроса, чем 3 сразу.

**Варианты > открытые вопросы**
```
❌ "Что ты хочешь получить?"
✅ "Результат: A (отчёт PDF), B (дашборд) или C (API)?"
```

**YAGNI (You Aren't Gonna Need It)**
Убирать фичи "может пригодится". Оставлять только необходимое сейчас.

```
❌ "Добавим экспорт в 5 форматов на будущее"
✅ "Начнём с одного формата, остальные добавим если понадобится"
```

**Конкретика > абстракция**
```
❌ "Можно сделать сайт"
✅ "Лендинг на Astro, 3 секции (hero, features, CTA), хостинг Vercel"
```

**Быстро к сути**
Штурм не должен быть долгим. 3-5 минут диалога максимум.

## 💡 Примеры использования

**Пример 1: Контент для YouTube**

```
Пользователь: "Хочу снять видео про автоматизацию"

Агент:
🔍 КОНТЕКСТ
Проверил аналитику: лучше заходят видео-кейсы с цифрами 
(просмотры +40% vs теория)

❓ УТОЧНЕНИЕ
Формат: A (туториал пошагово), B (кейс "сделал X за Y"), 
или C (сравнение инструментов)?

Пользователь: "B - кейс"

Агент:
💡 ТРИ ВАРИАНТА

Вариант 1: "Автоматизировал email за 2 часа" ⭐
Суть: Показать как настроил автоответы + сегментацию
Плюсы: Конкретный результат, повторяемо, есть цифры
Минусы: Узкая тема (email)
Время: 1 день (съёмка 3ч, монтаж 4ч)

Вариант 2: "5 автоматизаций которые сэкономили 15ч/нед"
Суть: Подборка разных кейсов
Плюсы: Широкий охват, больше value
Минусы: Менее детально каждый кейс
Время: 2 дня (материала больше)

Вариант 3: "От ручной работы к автопилоту за неделю"
Суть: История трансформации процесса
Плюсы: Эмоциональнее, вовлекает
Минусы: Сложнее структура
Время: 2-3 дня

🎯 РЕКОМЕНДАЦИЯ: Вариант 1
Конкретика + повторяемость = выше retention.
Остальные варианты можно сделать позже.

Какой выберешь?

Пользователь: "Первый"

Агент:
✅ ПЛАН

1. Подготовка (30 мин)
   - Скрипт: intro → проблема → решение → результат → outro
   - Тайм-коды: разметить 5 секций

2. Съёмка (2-3 часа)
   - Screencast настройки (1.5ч)
   - Talking head intro/outro (30 мин)
   - B-roll результаты (30 мин)

3. Монтаж (3-4 часа)
   - Склейка + переходы
   - Добавить графику (цифры, результаты)
   - Музыка + цветокор

4. SEO (30 мин)
   - Передать транскрипт в youtube-seo скил
   - Сгенерить title + description + tags

Одобряешь план?
```

## 🔧 Настройка под себя

### Для контент-криейторов

Перед штурмом агент может проверять:
- Аналитику каналов (что лучше заходит)
- Конкурентов (что делают другие)
- Стратегию канала (если есть файл)

Фильтровать идеи через:
- Целевую аудиторию
- Формат который лучше работает (видео/текст/кейс)
- Сложность производства vs ожидаемый результат

### Для разработчиков

Фильтровать через:
- Уже есть готовое решение?
- Будет использоваться >1 раза?
- Можно ли сделать проще?
- Что сломается если не сделать?

### Для автоматизаций

Фильтровать через:
- Set-and-forget возможно?
- Окупится ли время на настройку?
- Есть ли готовые инструменты (скилы, n8n templates)?

### Настройка детализации

По умолчанию агент масштабирует детализацию:
- Простая задача (< 30 мин) = краткий план
- Средняя (1-3ч) = пошаговый план
- Сложная (> 3ч) = разбивка на фазы

Можете явно запросить:
```
"Дай максимально детальный план" → подробная разбивка
"Дай общий plan" → high-level overview
```

### Интеграция с другими скиллами

**С `business-architect`:**
```
brainstorming генерит варианты → business-architect 
валидирует через unit economics и рынок
```

**С `deep-research-pro`:**
```
brainstorming определяет что нужно узнать → deep-research-pro 
глубоко исследует тему
```
