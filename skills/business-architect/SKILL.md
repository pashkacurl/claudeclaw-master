# Бизнес-Архитектор - От идеи к деньгам

## 📦 Установка

1. Скопируйте папку `business-architect/` в `skills/` вашего workspace
2. OpenClaw автоматически подхватит скилл при следующем сообщении
3. Проверка: скажите агенту "у меня есть идея..." или "как заработать на..."

## 🎯 Что умеет

- Валидировать бизнес-идеи через жесткие вопросы
- Искать боли рынка через Reddit, X, отзывы
- Анализировать конкурентов (цена, сильные/слабые стороны)
- Считать unit economics (LTV, CAC, margins)
- Составлять Lean Canvas на одной странице
- Проектировать MVP за 1-2 недели
- Создавать пошаговый план запуска с конкретными действиями

## ⚙️ Как работает

### Принципы

1. **Честность > вежливость.** Идея плохая - скажет прямо и объяснит почему
2. **Конкретика > абстракция.** Не "можно было бы", а "делай вот это"
3. **Деньги > интерес.** Не "интересный проект", а "вот так принесёт $X"
4. **Действия > анализ.** Максимум 30% времени на анализ, 70% на план действий
5. **Маленькие шаги > большие прыжки.** MVP за 1-2 недели, не за 3 месяца

### Рабочий процесс

#### Фаза 1: Приём идеи

Когда вы озвучиваете идею (даже сырую), агент:

1. Переформулирует её в одно предложение: "Ты хочешь [продукт] для [аудитория] который решает [боль] за [деньги]"
2. Задаст максимум 5 жёстких вопросов:
   - Кто конкретно за это заплатит?
   - Сколько они сейчас платят за решение этой проблемы?
   - Почему они выберут тебя, а не существующее решение?
   - Как ты будешь продавать (какие каналы)?
   - Сможешь ли ты сделать MVP за 2 недели?

#### Фаза 2: Исследование болей

Агент использует `web_search` и `web_fetch` для поиска:
- Reddit: тематические сабреддиты
- X/Twitter: жалобы по теме
- Отзывы конкурентов (негативные - там боли)
- Google Trends: динамика спроса

**Формат вывода:**
```
🔍 БОЛИ РЫНКА

1. [Боль] - частота упоминаний: X
   Цитата: "..."
   Источник: [ссылка]

2. [Боль] - частота: X
   ...

💡 ВОЗМОЖНОСТЬ: [что из этого следует]
```

#### Фаза 3: Анализ конкурентов

Найдет 3-5 конкурентов. Для каждого:

```
⚔️ КОНКУРЕНТЫ

| Название | Цена | Сильные стороны | Слабые стороны | Отзывы |
|----------|------|-----------------|----------------|--------|
| ...      | ...  | ...             | ...            | X/5 ⭐ |

🎯 ТВОЁ ПРЕИМУЩЕСТВО: [что можешь сделать лучше/дешевле/проще]
```

#### Фаза 4: Unit Economics

Считает просто, без MBA-терминов:

```
💰 ЭКОНОМИКА

Цена продукта: $X
Себестоимость (API, хостинг, время): $Y
Прибыль с 1 продажи: $Z

Чтобы выйти на $10K/мес нужно: N продаж
При конверсии сайта 2-3%: нужно M посетителей

ВЕРДИКТ: [реально / нужно доработать / нереально]
```

**Формулы:**
- **LTV** (Lifetime Value) = Средний чек × Кол-во покупок за жизнь клиента
- **CAC** (Customer Acquisition Cost) = Затраты на маркетинг / Кол-во новых клиентов
- **LTV/CAC** > 3 = хороший бизнес
- **Payback period** = CAC / (Месячный доход с клиента)
- **Маржа** = (Цена - Себестоимость) / Цена × 100%

#### Фаза 5: Lean Canvas

Одна страница, всё по делу:

```
📋 LEAN CANVAS

ПРОБЛЕМА:         [3 главные боли]
РЕШЕНИЕ:          [3 ключевые фичи MVP]
МЕТРИКИ:          [что измеряем]
ПРЕИМУЩЕСТВО:     [что сложно скопировать]
КАНАЛЫ:           [как узнают о продукте]
АУДИТОРИЯ:        [кто платит]
СТРУКТУРА ЗАТРАТ: [на что тратим]
ПОТОКИ ДОХОДА:    [откуда деньги]
```

#### Фаза 6: MVP и План запуска

```
🚀 ПЛАН ЗАПУСКА

MVP (2 недели):
День 1-3: [конкретные действия]
День 4-7: [конкретные действия]
День 8-10: [конкретные действия]
День 11-14: [тест + первые пользователи]

ПОСЛЕ MVP:
Неделя 3: [что делаем]
Неделя 4: [что делаем]
Месяц 2: [масштабирование]
```

### Стоп-сигналы

Агент остановит вас и скажет прямо, если:
- Идея требует команды > 1 человека для MVP
- Рынок слишком маленький (< $1M/год)
- Уже 10+ сильных конкурентов без явного преимущества
- MVP займёт > 1 месяца
- Маржа < 50% (скорее всего не окупится)
- LTV/CAC < 3 (убыточный бизнес)

## 💡 Примеры использования

**Пример 1: Онлайн-курс**

```

Агент:

Ты хочешь создать видеокурс по автоматизации для фрилансеров

2. Что они сейчас делают вместо курса? (YouTube, пробуют сами, 
   платят подрядчикам $500+)
3. Почему выберут тебя? (ты показываешь свою систему изнутри, 
   не теорию)
4. Как будешь продавать? (YouTube органика + email-лист)
5. MVP за 2 недели? (5 видео по 15 мин + лендинг - да)

💰 ЭКОНОМИКА
Цена: $49
Себестоимость: $10 (платформа, email-сервис)
Прибыль: $39
Для $5K/мес: 128 продаж
При конверсии 2%: 6,400 визитов на лендинг

🚀 ПЛАН MVP (14 дней)
День 1-3: Записать 5 видео (intro, setup, 3 кейса)
День 4-5: Лендинг на Carrd/Gumroad
День 6-7: Email-серия welcome (3 письма)
День 8-10: Бета-тест с 10 людьми из аудитории
День 11-14: Фикс по отзывам + запуск

✅ ВЕРДИКТ: Реально. Начинай.
```

## 🔧 Настройка под себя

### Адаптация под тип бизнеса

**Для контент-криейторов:**
- Фокус на органические каналы (YouTube, SEO, Telegram)
- Низкий CAC
- Проверка: есть ли аудитория сейчас?

**Для разработчиков:**
- Фокус на SaaS/инструменты
- Проверка: можно ли сделать MVP за 2 недели?
- Учитывать себестоимость API

**Для фрилансеров:**
- Фокус на продуктизацию услуг
- Проверка: можно ли продавать без личных встреч?

### Изменение целей дохода

По умолчанию скилл считает для $5-10K/мес. Можно изменить:

```
"Посчитай unit economics для цели $2K/мес"
"Сколько нужно клиентов для $20K/мес?"
```

### Настройка стоп-сигналов

Можете добавить свои критерии отсева:
- "Не рассматривай идеи требующие инвестиций > $1K"
- "Только B2C, без B2B"
- "Только пассивный доход (без активного времени)"

### Источники исследований

По умолчанию используются:
- Reddit (основной источник болей)
- X/Twitter (тренды и жалобы)
- Google Trends (динамика спроса)
- Product Hunt (конкуренты)
- Review-сайты (Trustpilot, G2, Capterra)

Можете добавить:
- Specific subreddits для вашей ниши
- Facebook groups
- LinkedIn posts
- YouTube comments
