# Deep Research Pro 🔬 - Глубокий ресёрч

## 📦 Установка

1. Скопируйте папку `deep-research-pro/` в `skills/` вашего workspace
2. OpenClaw автоматически подхватит скилл при следующем сообщении
3. Проверка: скажите агенту "проведи ресёрч по теме..."

## 🎯 Что умеет

- Системный сбор информации из 10+ источников
- Разбивка темы на подвопросы для полного покрытия
- Глубокое чтение ключевых источников (не только сниппеты)
- Синтез находок в структурированный отчёт
- Цитирование каждого утверждения со ссылкой
- Перекрёстная проверка фактов

## Когда использовать

✅ Подходит:
- "Проведи ресёрч по теме X"
- "Глубокий анализ рынка / технологии / конкурентов"
- "Что сейчас происходит с X? Собери полную картину"
- "Исследуй тему для видео / поста"
- Любая задача где нужен СИСТЕМНЫЙ сбор из 10+ источников

❌ Не использовать (есть инструменты лучше):
- Быстрый фактологический вопрос → `web_search`
- Саммари одного видео/статьи → скилл `summarize`
- Узкий запрос про один продукт → обычный поиск

## ⚙️ Как работает

### Workflow (6 шагов)

#### Step 1: Уточнение (30 сек)

Агент задаёт 1-2 вопроса:
- "Цель - узнать для себя, принять решение или написать контент?"
- "Какой угол или глубина нужна?"

Если "просто исследуй" → дефолтные настройки.

#### Step 2: Планирование

Разбить тему на 3-5 подвопросов.

**Пример:**
```
Тема: "AI агенты для бизнеса"

Подвопросы:
1. Какие основные платформы агентов существуют?
2. Какие реальные кейсы применения в бизнесе?
3. Сколько стоит внедрение?
4. Какие ограничения и риски?
5. Куда движется рынок в 2026?
```

#### Step 3: Мультиисточниковый поиск

Для КАЖДОГО подвопроса:
```
web_search(query="<ключевые слова подвопроса>", count=10)
```

**Стратегия:**
- 2-3 вариации ключевых слов на подвопрос
- Цель: 15-30 уникальных источников
- Приоритет: академические, официальные, авторитетные СМИ > блоги > форумы
- Для свежих тем: добавить `freshness="pm"` (последний месяц)

#### Step 4: Глубокое чтение ключевых источников

Для 3-5 лучших URL:
```
web_fetch(url="<url>", maxChars=5000)
```

**Важно:** Не полагаться только на сниппеты поиска - читать полные статьи.

#### Step 5: Синтез и отчёт

```markdown
# [Тема]: Отчёт Deep Research
*Дата: [дата] | Источников: [N] | Достоверность: [Высокая/Средняя/Низкая]*

## Краткое резюме
[3-5 предложений с ключевыми находками]

## 1. [Первая тема]
[Находки с цитатами]
- Ключевой факт ([Источник](url))
- Подтверждающие данные ([Источник](url))

## 2. [Вторая тема]
...

## Ключевые выводы
- [Вывод 1]
- [Вывод 2]
- [Вывод 3]

## Источники
1. [Название](url) - краткое описание
2. ...

## Методология
Исследовано [N] запросов. Проанализировано [M] источников.
Подвопросы: [список]
```

#### Step 6: Доставка

- Короткие темы → полный отчёт в чат
- Длинные отчёты → резюме + выводы в чат, полный отчёт сохранить в файл

**Путь сохранения:**
```
research/[slug]/report.md
```

### Правила качества

1. **Каждое утверждение - со ссылкой.** Без исключений.
2. **Перекрёстная проверка.** Один источник = пометить как "не подтверждено".
3. **Свежесть.** Приоритет источникам за последние 12 месяцев.
4. **Честность пробелов.** Не нашёл инфу - так и скажи.
5. **Без галлюцинаций.** Не знаешь - "недостаточно данных".

## 🔧 Настройка под себя

### Глубина исследования

По умолчанию: 15-30 источников, 3-5 подвопросов.

Можете запросить:
```
"Быстрый ресёрч за 5 минут" → 5-10 источников, 2-3 подвопроса
"Максимально глубокий анализ" → 30-50 источников, 6-8 подвопросов
```

### Фокус источников

По умолчанию: микс академических, СМИ, блогов.

Можете указать:
```
"Только академические источники"
"Фокус на практические кейсы, не теорию"
"Приоритет русскоязычным источникам"
```

### Формат вывода

По умолчанию: Markdown отчёт.

Альтернативы:
```
"Дай в формате для презентации" → bullet points
"Таблица сравнения" → table format
"Только выводы без деталей" → executive summary
```

### Сохранение

По умолчанию: `research/[topic-slug]/report.md`

Можете указать:
```
"Сохрани в memory/"
"Не сохраняй, только в чат"
"Сохрани как PDF" (если есть nano-pdf скилл)
```

### Запуск как субагент

Для длинных исследований (20+ минут):
```
"Запусти ресёрч в фоне и сообщи когда готово"
```

Агент создаст субагента:
```
sessions_spawn(
  task: "Deep research по теме [ТЕМА]. Сохрани в research/[slug]/report.md",
  label: "research-[slug]",
  model: "sonnet"
)
```

### Интеграция с другими скиллами

**С `business-architect`:**
```
deep-research-pro исследует рынок/конкурентов → 
business-architect использует данные для валидации идеи
```

**С `brainstorming`:**
```
brainstorming определяет что нужно узнать → 
deep-research-pro глубоко исследует тему
```

**С `copywriter`:**
```
deep-research-pro собирает факты и цитаты → 
copywriter пишет контент на основе исследования
```
